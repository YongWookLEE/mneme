# UI Guide

## 제품 톤

차분하고 도구적. Heirmos·Linear·Vercel 대시보드 계열의 미니컬한 다크 테마. 정보 밀도는 중간(폴더 트리·검색 결과를 한 화면에서 훑을 수 있도록). 장식 요소(그림자, 그라데이션, 일러스트) 최소.

## 디자인 참고 이미지

- `docs/design/Mneme-dashboard.png` — 대시보드 **레이아웃·정보 배치** 레퍼런스(2026-06-27 사용자 제공). 4개 stat 카드·오늘의 개요·빠른 작업·관계 맵 미리보기·AI 인사이트·최근 메모리·활동 로그·시스템 상태 배치만 참고. 색조는 모노크롬 정책으로 다시 매핑 — **이미지의 보라 강조는 채택하지 않음**
- `docs/design/Mneme-map.png` — `/map` 레이아웃 레퍼런스(2026-06-27 사용자 제공). 사이드바·상단바·범례·우측 패널 배치만 참고. **그래프 자체의 보라 강조는 채택하지 않음** — 그래프 톤은 모노크롬(중립 검정 배경 + 흰/회색 노드, 사용자 추가 지시 2026-06-27)
- 위 이미지는 "참고용". 픽셀 단위 모사 아님. 톤·구조·정보 배치 우선, 세부는 phase 11/17 구현 시 결정

## 레이아웃 원칙

- **고정 좌측 사이드바**(폭 ~240px): 로고 M(흰→연실버 그라데이션, 또는 단색 `#E6E7E8`. 보라/색조 사용 안 함) + 아이콘+라벨 메뉴(폴더/검색/맵/대시보드/인사이트/태그/설정) + 하단에 오늘 토큰 사용 게이지(progress bar) + 피드백 보내기 버튼
- **상단바**(높이 ~56px): 좌측 breadcrumb(현재 페이지 + 서브 드롭다운, 예: `맵 ▸ 그래프 뷰 ▾`) + 중앙 단일 검색바(`Cmd/Ctrl+K` 힌트 inline) + 우측 도움말·알림·프로필 메뉴
- 메인 영역은 페이지별 3분할 또는 단일 캔버스
- 모달은 최소화. 폼은 인라인 또는 사이드 패널
- 모바일 대응은 후순위(검색·읽기만 부분 지원, 편집은 데스크톱 권장)

## 색상 (다크 테마, 중립 모노크롬)

전체 톤은 **중립 그레이 스케일**. 푸른 기·따뜻한 기 모두 배제. 강조는 색조가 아니라 **밝기 차**로 만든다(흰색 ↔ 회색). 의미가 명확한 상태(위험/성공/경고)에만 색조를 허용한다.

- 배경: `#0E0E10` (중립 검정)
- 표면: `#16171A` (카드/패널)
- 표면(은은히 밝게): `#1C1D20` (호버, 활성 행)
- 텍스트: `#E6E7E8` (본문) / `#9B9CA0` (보조) / `#5B5C60` (비활성)
- 경계선: `#22232A` (1px, 매우 미세)
- 강조: 색조 없이 텍스트색(`#E6E7E8`) 자체. 활성 메뉴 좌측 막대·검색 포커스 링·1차 버튼 텍스트 등에 사용
- 강조 glow(/map 선택 노드): 순백 `#FFFFFF` 22% 외광 — 색조 없음
- 위험: `#E5484D` (폐기 / 충돌 / 깨진 링크 / 한도 초과)
- 성공: `#48BB78` (저장 완료, 상승 지표 `↑`)
- 경고: `#F7B955` (사용량 임계치, 토큰 게이지)
- 신뢰도 배지: 표면색 칩 + 보조 텍스트(`신뢰도 94%`). 색조 없음

## 타이포그래피

- 제목: Inter / Pretendard, 600, 18~24px
- 본문: Inter / Pretendard, 400, 14~15px, line-height 1.6
- 코드/키: JetBrains Mono, 13px
- 보조 텍스트: 13px, `#9BA3AD`

## 컴포넌트 패턴

- 버튼: shadcn/ui `Button` — 1차(`default`) / 2차(`outline`) / 위험(`destructive`) 3종만
- 카드: 경계선 1px + 표면색, 둥근 모서리 12~16px, 패딩 20~24px. 깊은 그림자 없음
- Stat 카드: 큰 숫자(28~32px, 600) + 라벨(13px, 보조 텍스트) + 우측 아이콘(원형 배경, 강조색 16% opacity) + 하단 delta(`↑ 12.4% 지난 7일 대비`, 상승은 성공색 / 하락은 위험색)
- 폼: 라벨은 입력 상단, 에러는 입력 아래 빨강. 인라인 검증
- 리스트: 항목 간 1px divider, 호버 시 표면색 살짝 밝게
- 사이드바 메뉴 항목: 아이콘(20px) + 라벨(14px). 활성 항목은 본문색 텍스트(`#E6E7E8`) + 좌측 4px 흰색 막대 + 표면색 배경 살짝
- 토큰 사용 게이지: 사이드바 하단. `Tokens 812,345 / 1,000,000 (81%)` + 1px 높이 progress bar. 평소는 `#E6E7E8` 흰색, 80% 이상 경고색(`#F7B955`), 100% 도달 시 위험색(`#E5484D`) + 메시지 변경
- Stat 카드 아이콘 원형 배경: 텍스트색 10% opacity(즉 `#E6E7E81A`). 아이콘 자체는 본문색. 채도 강한 색조 사용 금지
- 1차 버튼(`default`): 표면 `#1C1D20` + 본문색 텍스트 + 1px `#33343A` 경계. 호버 시 표면 한 단계 밝게. 강조 보라 사용 안 함
- 검색 포커스 링: 흰색(`#E6E7E8`) 1px outline + 배경 살짝 밝아짐
- Sparkle/AI 인사이트 아이콘: 흰색. 채도 강한 색조 사용 금지
- 신뢰도 배지: 표면색 칩 + 보조 텍스트(`신뢰도 94%`). 색조 없음
- 마크다운 뷰어: GitHub 스타일. 코드 블록은 어두운 톤, 언어 자동 감지. **`[[link]]`는 강조색 링크로 렌더**, 깨진 링크는 회색 점선 + 경고 아이콘
- 마크다운 편집기: Monaco 또는 CodeMirror 6, 단축키 통일, 미리보기 토글. `[[` 입력 시 메모리 제목 자동완성 팝업(임베딩 유사도 + 최근 메모리 정렬)
- 관계 그래프(/map): force-directed 레이아웃. 강조색 노드 = 현재 메모리, 보조색 = 1-hop, 비활성색 = 2-hop+. 깨진 링크 = 위험색 점선. 라이브러리는 phase 진입 시 ADR로 결정(react-force-graph / Cytoscape.js / sigma 후보)
- 폴더 트리: 들여쓰기 16px, chevron, 현재 선택 항목 강조색 좌측 바, 드래그앤드롭 지원
- 검색 결과 카드: 제목 + 요약 2줄 + 폴더 경로 + 태그 + 점수 막대

## 페이지 구성

- `/login`: Google 로그인 버튼 + 제품 한 줄 소개
- `/onboarding`: 첫 로그인 4단계 투어 (메모리 만들기 → 키 발급 → 클라이언트 연결 → 검색). 건너뛰기 가능, `/settings`에서 재시작
- `/`: 대시보드. 좌측 사이드바는 전역 공통. 메인은 다음 블록 — (1) 상단 4개 stat 카드(`전체 메모리`/`오늘 저장`/`연결 수`/`검색 정확도`) — 큰 숫자 + 라벨 + 지난 7일 대비 delta + 아이콘 원형 배경, (2) 좌측 컬럼: `오늘의 개요` 카드(sparkle 아이콘 + 한 문단 + 핵심 주제/활발한 영역/추천 액션 bullet) → `최근 메모리` 테이블(탭: 전체/문서/노트/이미지/코드, 컬럼: 제목/경로/태그 칩/수정 시간) → `활동 로그`(아이콘 + 한 줄 + 시간), (3) 우측 컬럼: `빠른 작업` 2×2(새 메모리/검색 시작/관계 맵 열기/API 키 관리) → `관계 맵 미리보기`(미니 그래프 + 범례, "전체 맵 열기" 링크) → `AI 인사이트` 카드(sparkle + 문장 + 신뢰도 배지). (4) 하단: `시스템 상태`(저장소·인덱싱·검색·API 정상/주의/오류). 메모리 편집 화면은 별도(/m/:id) 또는 사이드 패널 — 본문 안 `[[link]]` 클릭 시 이동, 메타 영역에 backlink 목록
- `/map`: 본문 `[[link]]` 기반 관계 그래프. **그래프 자체는 완전 모노크롬**(흰/회색 스케일, 색조 사용 안 함). 노드 = 메모리, 엣지 = `[[link]]`. 레이아웃 — 좌상단 "지식 맵 ▾" 드롭다운, 우상단 토글 4종(`필터` / `레이아웃` / `깊이 N` / `톱니바퀴`), 좌하단 컨트롤(`+` / `−` / 전체화면 / 중앙 복귀), 좌하단 범례 카드(노드 유형/연결 유형/그래프 통계), 하단 가운데 인터랙션 힌트(`드래그 이동` / `스크롤 확대·축소` / `더블클릭 중심`). 노드 색 정책 — 기본 `#9B9CA0`(회색), 큰(연결 많은) 노드는 더 큼 + 약간 더 밝음 `#C8C9CC`, 호버는 `#E6E7E8`, **선택은 순백 `#FFFFFF` + 외광 glow**. 엣지는 `#3A3B40` 1px, 선택 노드의 1-hop 엣지만 `#9B9CA0`. 깨진 링크는 위험색 점선·경고 아이콘. 우측 패널: 선택된 메모리 — 제목 + 경로(breadcrumb) + 태그 칩 + 요약 문단 + `연결 정보` 4개 stat(직접 / 간접 / 참조됨 / 참조함) + `연결 메모 (N)` 리스트(각 행: 제목, 경로, 직접/간접 배지, ⋮ 액션) + "메모 열기" CTA
- `/archive`: archive(`mn_archive`) 된 메모리 일괄 보기. 항목 행에 "복구" 버튼(`mn_restore`). 같은 경로에 활성 메모리 존재 시 버튼 비활성 + 사유 툴팁
- `/keys`: API 키 발급/목록/폐기/회전 + 마지막 사용 시각 + 일일 사용량. **MCP 명령 빌더** — 키 발급 직후 패널에서 클라이언트 토글(Claude Code / Codex) + scope 토글(local / user / project) → `claude mcp add --transport http mneme https://<host>/mcp --header "Authorization: Bearer mn_..." --scope <scope>` 명령 표시 + 복사 버튼. 평문 키는 화면 새로고침 시 사라짐
- `/guide`: 클라이언트별 연결 가이드 (Claude.ai, ChatGPT 개발자 모드, Codex CLI, REST 예제, 스크린샷)
- `/settings`: 계정 정보, 토큰 사용량, 활동(감사 로그), 데이터 export/import, 계정 삭제
- `/activity`: 본인 감사 로그 (테이블, 필터: 액션/날짜)

## 상태별 UI 가이드

### 빈 상태 (Empty)
- 메모리 0개: 단순 아이콘 + "첫 메모리를 만들거나 AI에게 시키세요" + 두 액션(메모리 작성, 가이드 보기)
- 검색 결과 0개: 검색어 표시 + "다른 키워드 시도" + 최근 메모리 5개 제시
- 폴더 비어있음: "이 폴더는 비어있습니다" + 메모리 추가 버튼
- 키 0개: 온보딩 흐름으로 안내

### 로딩
- 200ms 이내 응답 기대: 스피너 없이
- 200ms~2s: 인라인 스피너 (작게, 위치 보존)
- 2s 초과 가능 작업(메모리 저장/임베딩, export, import): 진행 표시 + 예상 시간 안내
- 스켈레톤 UI는 폴더 트리·메모리 리스트 첫 로드에만

### 에러
- 인라인 에러(폼 검증): 입력 아래 빨강 메시지
- 페이지 에러(서버 5xx): 빈 상태 + 메시지 + 재시도 버튼 + trace_id 표시
- 토스트: 비차단 알림(저장 완료, 키 복사 성공)에만. 에러 토스트는 자동 사라지지 않음(닫기 필수)
- 401: 자동 `/login` 리다이렉트 + 토스트
- 409 (낙관적 락 충돌): 모달로 두 버전 diff 표시 → 사용자 선택(내 버전 / 서버 버전 / 병합)
- 429: "잠시 후 다시 시도" + 남은 대기 초 (Retry-After 헤더 활용)
- 토큰 한도 초과 (429 ERR_QUOTA_EXCEEDED): 위험색 배너 + "내일 자정 UTC 리셋"

### 위험 동작 확인
- 메모리 archive: 인라인 undo 토스트 5초 (자동 archive 후 되돌리기)
- 메모리 영구 삭제: **MVP에선 노출하지 않음**. soft delete + restore가 약속
- API 키 폐기: 다이얼로그(키 이름 입력으로 재확인) + 폐기 후 즉시 401 안내
- 계정 삭제: 이메일 입력 확인 + 30일 유예 안내 + 그 사이 복구 가능 명시
- 폴더 이동(영향 대규모): "이 폴더의 메모리 N개 경로가 변경됩니다" 확인
- import 충돌: 항목별 선택(건너뜀/덮어쓰기/이름 변경) + 일괄 적용 버튼

### 사용량 임계치
- 일일 토큰 80% 도달: 경고색 배너 ("오늘 한도의 80% 사용됨")
- 100% 도달: 위험색 배너 + 호출 차단 + "내일 00:00 UTC 리셋"
- 키 30일 이상 미사용: 키 목록에 회색 배지 + 회전 권장 툴팁

## 검색·필터·정렬

- 검색바: 입력 즉시 디바운스 200ms 후 hybrid search
- 필터(검색 결과 상단 칩):
  - 폴더: 트리에서 선택 또는 경로 입력
  - 태그: 자동완성 다중 선택
  - 날짜: 7일 / 30일 / 90일 / 사용자 지정
  - 아카이브 포함 (기본 제외)
- 정렬:
  - 관련도 (기본, 하이브리드 점수)
  - 최신 수정순
  - 최신 생성순
  - 제목 가나다순
- 검색 결과 카드 호버: 본문 첫 5줄 미리보기
- 결과 클릭: 메모리 본문 열기 (히스토리 push, 뒤로가기로 검색 결과 복귀)

## 본문 [[link]] UX

- 표기: `[[메모리 제목]]` 또는 `[[mem_<base32>]]`. 제목 중복 시 후자 권장
- 편집기 자동완성: `[[` 입력 시 임베딩 유사도 + 최근 수정 순으로 최대 8개 후보. 키보드 ↑↓ 선택, Enter 확정
- 뷰어 렌더: 강조색 텍스트 + 호버 시 대상 메모리 제목·요약 카드 (200ms 지연). 클릭 시 대상 메모리 열기
- 깨진 링크(`[[없는 제목]]`): 회색 점선 + 경고 아이콘. 호버 툴팁 "이 제목의 메모리가 없습니다 — 만들기" + 클릭 시 새 메모리 작성(제목 미리 채움)
- Backlink 패널: 메모리 우측 메타 영역 하단에 "이 메모리를 참조하는 메모리" 목록. 항목 클릭 시 이동
- 자동 추천: 본문 아래 "관련 가능성 있는 메모리"(임베딩 코사인 상위 N개). "본문에 [[link]]로 추가" 버튼 → 현재 커서 위치에 삽입

## 온보딩 흐름 (첫 로그인)

1. **환영 + 컨셉**: "AI들이 공유하는 당신의 기억" + 한 줄 데모(GIF 또는 텍스트)
2. **첫 메모리 만들기**: 예시 텍스트 미리 채워진 입력 (수정 후 저장 또는 그대로 저장). 저장 완료 후 분류 결과 시연
3. **API 키 발급**: 1개 자동 생성, 1회 노출 + 복사 버튼 + "복사했나요?" 체크박스
4. **연결 가이드 링크**: 사용 중인 AI 도구 선택 → 해당 가이드로 이동. "나중에" 선택 시 종료
5. **건너뛰기 가능**. `/settings`에서 언제든 재시작

## i18n 전략

- 기본 언어: 한국어. 영문 UI는 MVP 제외(코드 구조는 `react-i18next`로 분리하되 문구는 한국어만 입력)
- 네임스페이스: common, dashboard, keys, guide, settings, errors, onboarding
- `users.locale` 컬럼에 사용자 언어 저장 (기본 `ko`)
- 에러 메시지: 백엔드는 코드(`ERR_RATE_LIMIT`), 프론트가 번역
- 시간 표시: `Intl.DateTimeFormat`으로 사용자 로컬, 데이터는 UTC

## 접근성

- 모든 인터랙티브 요소는 키보드 포커스 가능, 포커스 링 명시
- 색상만으로 정보 전달 금지 (위험/성공/경고는 아이콘 병기)
- 최소 텍스트 대비 4.5:1 (다크 테마 기준 검증)
- 검색바·폴더 트리·본문 영역 간 Tab 순서 일관
- 스크린 리더 ARIA: 폴더 트리 펼침/접힘, 검색 결과 카운트, 토스트 알림
- 모달은 포커스 트랩 + Esc 닫기

## 모바일 부분 대응

- 768px 이하: 폴더 트리 숨김(햄버거 아이콘), 검색·읽기 우선
- 메모리 편집은 데스크톱 권장 메시지 (모바일에선 본문 텍스트 영역만 단순 편집)
- 키 발급/폐기는 가능 (긴 키 복사 UX 신경 써서)

## 빈 디자인 자원 자리

- 로고: 별도 디자인 전까지 텍스트 "Mneme" + 작은 점 마크
- 일러스트: 사용 안 함
- 스크린샷: `/guide` 페이지에 placeholder div, 후속 phase에서 교체

## 금지 패턴

- 자동 재생 애니메이션, 의미 없는 모션. 이유: 도구의 피로감
- 영구 삭제 버튼 상시 노출. 이유: soft delete 약속
- 한 화면에 3개 초과 강조색 사용. 이유: 우선순위 붕괴
- 모달 중첩. 이유: 흐름 단절
- 자동 사라지는 에러 토스트. 이유: 사용자가 놓치면 디버깅 불가
- API 키 새로고침 후 평문 재노출. 이유: 1회 노출 보안 약속 위반
- 사용자 입력 검증을 클라이언트에서만 처리. 이유: 서버 검증이 진실
- 로딩 스피너 없이 5초 이상 대기. 이유: 응답 없음과 구분 안 됨
- 검색 결과에 본문 전체 노출. 이유: 토큰·렌더링 비용, 본문은 read로 별도 조회
