# UI Guide

## 제품 톤

차분하고 도구적. Heirmos·Linear·Vercel 대시보드 계열의 미니멀 다크 테마. 정보 밀도는 중간(폴더 트리·검색 결과를 한 화면에서 훑을 수 있도록). 장식 요소(그림자, 그라데이션, 일러스트) 최소.

## 레이아웃 원칙

- 3분할 기본: 좌측 폴더 트리 / 중앙 메모리 본문(마크다운) / 우측 메타·관계
- 상단 단일 검색바가 모든 페이지에서 일관 위치, `Cmd/Ctrl+K`로 어디서든 호출
- 모달은 최소화. 폼은 인라인 또는 사이드 패널
- 모바일 대응은 후순위(검색·읽기만 부분 지원, 편집은 데스크톱 권장)

## 색상 (다크 테마)

- 배경: `#0B0D10` (거의 검정)
- 표면: `#14171B` (카드/패널)
- 텍스트: `#E6E8EB` (본문) / `#9BA3AD` (보조) / `#5B6271` (비활성)
- 강조: `#7C9CFF` (링크/포커스)
- 경계선: `#1F242B`
- 위험: `#E5484D` (폐기, 충돌)
- 성공: `#48BB78` (저장 완료, 키 발급)
- 경고: `#F7B955` (사용량 임계치 근접)

## 타이포그래피

- 제목: Inter / Pretendard, 600, 18~24px
- 본문: Inter / Pretendard, 400, 14~15px, line-height 1.6
- 코드/키: JetBrains Mono, 13px
- 보조 텍스트: 13px, `#9BA3AD`

## 컴포넌트 패턴

- 버튼: shadcn/ui `Button` — 1차(`default`) / 2차(`outline`) / 위험(`destructive`) 3종만
- 카드: 경계선 1px + 표면색, 둥근 모서리 8px, 패딩 16~20px
- 폼: 라벨은 입력 상단, 에러는 입력 아래 빨강. 인라인 검증
- 리스트: 항목 간 1px divider, 호버 시 표면색 살짝 밝게
- 마크다운 뷰어: GitHub 스타일. 코드 블록은 어두운 톤, 언어 자동 감지
- 마크다운 편집기: Monaco 또는 CodeMirror 6, 단축키 통일, 미리보기 토글
- 폴더 트리: 들여쓰기 16px, chevron, 현재 선택 항목 강조색 좌측 바, 드래그앤드롭 지원
- 검색 결과 카드: 제목 + 요약 2줄 + 폴더 경로 + 태그 + 점수 막대

## 페이지 구성

- `/login`: Google 로그인 버튼 + 제품 한 줄 소개
- `/onboarding`: 첫 로그인 4단계 투어 (메모리 만들기 → 키 발급 → 클라이언트 연결 → 검색). 건너뛰기 가능, `/settings`에서 재시작
- `/`: 대시보드 (폴더 트리 + 미리보기/편집 + 검색)
- `/keys`: API 키 발급/목록/폐기/회전 + 마지막 사용 시각 + 일일 사용량
- `/guide`: 클라이언트별 연결 가이드 (Claude.ai, ChatGPT 개발자 모드, Codex CLI, REST 예제, 스크린샷)
- `/settings`: 계정 정보, 토큰 사용량, 활동(감사 로그), 데이터 export/import, 계정 삭제
- `/activity`: 본인 감사 로그 (테이블, 필터: 액션/날짜)

## 상태별 UI 가이드

### 빈 상태 (Empty)
- 메모리 0개: 단순 아이콘 + "첫 메모리를 만들거나 AI에게 시키세요" + 두 액션(메모리 작성, 가이드 보기)
- 검색 결과 0개: 검색어 표시 + "다른 키워드 시도" + 최근 메모리 5개 제시
- 폴더 비어있음: "이 폴더는 비어있습니다" + 메모리 추가 버튼
- 키 0개: 온보딩 흐름으로 안내

### 로딩
- 200ms 이내 응답 기대: 스피너 없이
- 200ms~2s: 인라인 스피너 (작게, 위치 보존)
- 2s 초과 가능 작업(메모리 저장/임베딩, export, import): 진행 표시 + 예상 시간 안내
- 스켈레톤 UI는 폴더 트리·메모리 리스트 첫 로드에만

### 에러
- 인라인 에러(폼 검증): 입력 아래 빨강 메시지
- 페이지 에러(서버 5xx): 빈 상태 + 메시지 + 재시도 버튼 + trace_id 표시
- 토스트: 비차단 알림(저장 완료, 키 복사 성공)에만. 에러 토스트는 자동 사라지지 않음(닫기 필수)
- 401: 자동 `/login` 리다이렉트 + 토스트
- 409 (낙관적 락 충돌): 모달로 두 버전 diff 표시 → 사용자 선택(내 버전 / 서버 버전 / 병합)
- 429: "잠시 후 다시 시도" + 남은 대기 초 (Retry-After 헤더 활용)
- 토큰 한도 초과 (429 ERR_QUOTA_EXCEEDED): 위험색 배너 + "내일 자정 UTC 리셋"

### 위험 동작 확인
- 메모리 archive: 인라인 undo 토스트 5초 (자동 archive 후 되돌리기)
- 메모리 영구 삭제: **MVP에선 노출하지 않음**. soft delete + restore가 약속
- API 키 폐기: 다이얼로그(키 이름 입력으로 재확인) + 폐기 후 즉시 401 안내
- 계정 삭제: 이메일 입력 확인 + 30일 유예 안내 + 그 사이 복구 가능 명시
- 폴더 이동(영향 대규모): "이 폴더의 메모리 N개 경로가 변경됩니다" 확인
- import 충돌: 항목별 선택(건너뜀/덮어쓰기/이름 변경) + 일괄 적용 버튼

### 사용량 임계치
- 일일 토큰 80% 도달: 경고색 배너 ("오늘 한도의 80% 사용됨")
- 100% 도달: 위험색 배너 + 호출 차단 + "내일 00:00 UTC 리셋"
- 키 30일 이상 미사용: 키 목록에 회색 배지 + 회전 권장 툴팁

## 키보드 단축키 (전역)

| 단축키 | 동작 |
|---|---|
| `Cmd/Ctrl+K` | 검색 패널 열기 |
| `Cmd/Ctrl+N` | 새 메모리 |
| `Cmd/Ctrl+S` | 현재 메모리 저장 |
| `Cmd/Ctrl+E` | 편집 모드 토글 |
| `Cmd/Ctrl+Shift+F` | 폴더 트리 포커스 |
| `Esc` | 모달 닫기 / 편집 취소 |
| `/` | 검색바 포커스 (입력창 외부에서) |
| `?` | 단축키 도움말 모달 |
| 폴더 트리: `↑/↓` | 항목 이동 |
| 폴더 트리: `→/←` | 펼치기/접기 |
| 폴더 트리: `Enter` | 선택 항목 열기 |

## 검색·필터·정렬

- 검색바: 입력 즉시 디바운스 200ms 후 hybrid search
- 필터(검색 결과 상단 칩):
  - 폴더: 트리에서 선택 또는 경로 입력
  - 태그: 자동완성 다중 선택
  - 날짜: 7일 / 30일 / 90일 / 사용자 지정
  - 아카이브 포함 (기본 제외)
- 정렬:
  - 관련도 (기본, 하이브리드 점수)
  - 최신 수정순
  - 최신 생성순
  - 제목 가나다순
- 검색 결과 카드 호버: 본문 첫 5줄 미리보기
- 결과 클릭: 메모리 본문 열기 (히스토리 push, 뒤로가기로 검색 결과 복귀)

## 온보딩 흐름 (첫 로그인)

1. **환영 + 컨셉**: "AI들이 공유하는 당신의 기억" + 한 줄 데모(GIF 또는 텍스트)
2. **첫 메모리 만들기**: 예시 텍스트 미리 채워진 입력 (수정 후 저장 또는 그대로 저장). 저장 완료 후 분류 결과 시연
3. **API 키 발급**: 1개 자동 생성, 1회 노출 + 복사 버튼 + "복사했나요?" 체크박스
4. **연결 가이드 링크**: 사용 중인 AI 도구 선택 → 해당 가이드로 이동. "나중에" 선택 시 종료
5. **건너뛰기 가능**. `/settings`에서 언제든 재시작

## i18n 전략

- 기본 언어: 한국어. 영문 UI는 MVP 제외(코드 구조는 `react-i18next`로 분리하되 문구는 한국어만 입력)
- 네임스페이스: common, dashboard, keys, guide, settings, errors, onboarding
- `users.locale` 컬럼에 사용자 언어 저장 (기본 `ko`)
- 에러 메시지: 백엔드는 코드(`ERR_RATE_LIMIT`), 프론트가 번역
- 시간 표시: `Intl.DateTimeFormat`으로 사용자 로컬, 데이터는 UTC

## 접근성

- 모든 인터랙티브 요소는 키보드 포커스 가능, 포커스 링 명시
- 색상만으로 정보 전달 금지 (위험/성공/경고는 아이콘 병기)
- 최소 텍스트 대비 4.5:1 (다크 테마 기준 검증)
- 검색바·폴더 트리·본문 영역 간 Tab 순서 일관
- 스크린 리더 ARIA: 폴더 트리 펼침/접힘, 검색 결과 카운트, 토스트 알림
- 모달은 포커스 트랩 + Esc 닫기

## 모바일 부분 대응

- 768px 이하: 폴더 트리 숨김(햄버거 아이콘), 검색·읽기 우선
- 메모리 편집은 데스크톱 권장 메시지 (모바일에선 본문 텍스트 영역만 단순 편집)
- 키 발급/폐기는 가능 (긴 키 복사 UX 신경 써서)

## 빈 디자인 자원 자리

- 로고: 별도 디자인 전까지 텍스트 "Mneme" + 작은 점 마크
- 일러스트: 사용 안 함
- 스크린샷: `/guide` 페이지에 placeholder div, 후속 phase에서 교체

## 금지 패턴

- 자동 재생 애니메이션, 의미 없는 모션. 이유: 도구의 피로감
- 영구 삭제 버튼 상시 노출. 이유: soft delete 약속
- 한 화면에 3개 초과 강조색 사용. 이유: 우선순위 붕괴
- 모달 중첩. 이유: 흐름 단절
- 자동 사라지는 에러 토스트. 이유: 사용자가 놓치면 디버깅 불가
- API 키 새로고침 후 평문 재노출. 이유: 1회 노출 보안 약속 위반
- 사용자 입력 검증을 클라이언트에서만 처리. 이유: 서버 검증이 진실
- 로딩 스피너 없이 5초 이상 대기. 이유: 응답 없음과 구분 안 됨
- 검색 결과에 본문 전체 노출. 이유: 토큰·렌더링 비용, 본문은 read로 별도 조회
